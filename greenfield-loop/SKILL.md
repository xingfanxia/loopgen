# greenfield-loop → /loopgen (greenfield)

**Superseded by [`/loopgen`](../loopgen/SKILL.md).** One skill now composes all
four loop archetypes from a single primitive vocabulary.

`/greenfield-loop X` is shorthand for **`/loopgen X` started from the `greenfield`
archetype**. loopgen still classifies the task and may diverge where it demands —
recording every divergence in its provenance preamble — so you get the green-field
discovery loop you expect, plus hybrid coverage when the task isn't purely
greenfield.

The greenfield loop's actual definition lives in loopgen:

- shape, defaults, forbidden divergences — `loopgen/archetypes/greenfield.md`
- emitted prompt body — `loopgen/templates/bodies/greenfield-body.md`
- the 11 green-field invariants — `loopgen/references/greenfield-invariants.md`

> Note: greenfield is the one archetype with no legacy prompt template — its body
> was reconstructed from this skill's emit procedure + the 11 invariants, so it is
> the least battle-tested of the four. Treat its emissions with a closer first read.
