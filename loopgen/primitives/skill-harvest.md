# skill-harvest (shared primitive)

## Purpose

The block that turns a real run's lessons into the **dogfooding citations** the
family's promotion-bar rule requires. Identical across all archetypes. The
frontier / goal / story bodies inline this block verbatim; the greenfield body
references it via `{{INCLUDE skill-harvest}}`. This file is the single source so
all four stay in sync.

## Include directive

`{{INCLUDE skill-harvest}}` in an archetype body resolves to the block below
(everything after the `---`). `{{ARTIFACT_LOCATIONS}}` in the block is filled
from the frontload audit at composition time.

## Why it is load-bearing

A run that surfaces a generalizable lesson and doesn't capture it lets that
lesson evaporate. Promotion of any new archetype / primitive / overlay requires
at least one such note. Without the
harvest mechanism the bar has no supply.

---

### Skill Harvest

When an iteration exposes a reusable process lesson — a failure mode the
skill didn't yet name, an invariant that would have prevented drift, a
pattern that generalizes beyond this specific run — write a **Skill
Harvest note** to the run's artifact directory:

- **target skill** — this skill, or a sibling / parent
- **observed gap** — the rule that's missing or under-specified
- **evidence iteration** — which iteration revealed it
- **proposed rule** — suggested wording for the patch
- **why it generalizes** — or note that it doesn't
- **suggested patch wording** — drop-in text
- **accidental-encouragement risk** — what bad behavior the new rule
  could enable

Harvest notes are the **dogfooding citations** the family's promotion-bar
rule requires. Without them, lessons from real runs
evaporate.
